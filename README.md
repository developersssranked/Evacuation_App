# Evacuation

Android-приложение, которое показывает, куда бежать, если что. Берёт твою текущую геопозицию, даёт расставить на карте точки укрытий и строит до ближайшего из них пешеходный маршрут.

Делалось как учебный проект, но вполне рабочее: тайлы карты честные (OpenStreetMap), маршруты считает публичный OSRM, координаты — Fused Location Provider от Google Play Services. Никаких платных API-ключей не требуется.

## Что умеет

- показывает карту OSM с твоим местоположением, обновляется в реальном времени;
- даёт поставить укрытие долгим нажатием по карте или кнопкой «Здесь» (по текущим координатам);
- по тапу на маркер строит маршрут от тебя до этого укрытия;
- по кнопке «К ближайшему» сама ищет ближайшее по прямой укрытие (формула гаверсинусов) и запрашивает у OSRM пешеходный маршрут;
- показывает длину маршрута и оценочное время в пути;
- весь интерфейс на русском.

## Стек

- Kotlin + Jetpack Compose (UI декларативный)
- MVVM, состояние через `StateFlow`
- Корутины для всего асинхронного
- [osmdroid](https://github.com/osmdroid/osmdroid) — рендеринг тайлов OpenStreetMap
- [OSRM](https://project-osrm.org/) — маршрутизация, профиль `foot`
- Google Play Services Location (Fused Location Provider)
- OkHttp для HTTP-вызовов к OSRM

minSdk 24, targetSdk 36, JDK 11.

## Структура

```
app/src/main/java/com/thirdapp/evacuation/
├── MainActivity.kt                 — точка входа, поднимает Compose-экран
├── EvacuationViewModel.kt          — состояние, оркестровка локации и маршрутов
├── model/Models.kt                 — Coord, Shelter, UserLocation, haversineMeters
├── location/LocationProvider.kt    — Fused Location → Flow<UserLocation>
├── network/OsrmClient.kt           — обёртка над OSRM, GeoJSON → список точек
└── ui/
    ├── EvacuationScreen.kt         — карта, маркеры, маршрут, панель управления
    └── theme/                      — цвета, типографика, тема Material 3
```

Идея простая: `LocationProvider` и `OsrmClient` про данные и ничего не знают про UI. `EvacuationViewModel` собирает их вместе и держит пять `StateFlow` (укрытия, локация, маршрут, флаг загрузки, ошибка). Compose-экран подписан на эти потоки и перерисовывается сам.

## Запуск

Нужны:
- Android Studio (Hedgehog или новее)
- JDK 17 (Android Studio обычно идёт с ним из коробки)
- Эмулятор API 24+ **с Google Play Services** — без них Fused Location Provider не работает. В AVD Manager бери образ с пометкой *Google Play* (не *Google APIs*, не *AOSP*).

Дальше:

1. Открой папку проекта в Android Studio, дай ему синкнуть Gradle. Первый раз тянет osmdroid, play-services-location, okhttp — пара минут.
2. Запусти эмулятор.
3. Нажми Run (Shift+F10).
4. На запуске приложение попросит доступ к геолокации — соглашайся, иначе карта будет, а маркера и кнопок маршрута — нет.

### Передать локацию в эмулятор

По умолчанию свежий эмулятор не знает, где он. Чтобы карта отцентрилась и появился маркер:

- три точки сбоку от окна эмулятора → **Extended Controls** → **Location**;
- введи координаты или найди точку через поиск, нажми **Send**;
- альтернатива через терминал: `adb emu geo fix <долгота> <широта>` (именно в таком порядке).

Для имитации движения по маршруту — там же вкладка **Routes**, можно загрузить GPX-файл и проиграть.

## Как пользоваться

1. Дай разрешение на локацию — на карте появится синий кружок (это ты).
2. Долгий тап в любую точку карты → ставится зелёный маркер укрытия. Или кнопка «Здесь» — поставит укрытие по твоим координатам.
3. Тап на маркер → строится маршрут от тебя до этого укрытия. Синяя линия поверх карты, в карточке сверху — длина и время.
4. Кнопка «К ближайшему» — то же самое, но укрытие выбирается само (ближайшее по прямой).
5. «Очистить» — стирает все укрытия и маршрут.

## Грабли, на которые я наступил

- **Тайлы не грузились на Android 10+.** osmdroid по умолчанию пишет тайл-кэш в external storage, а scoped storage его не пускает. Лечится явным указанием пути в `Configuration.getInstance().osmdroidBasePath` / `osmdroidTileCache` внутри `cacheDir` приложения.
- **403 от tile.openstreetmap.org.** OSM банит запросы с пустым или дефолтным User-Agent. Поставил `"Evacuation/1.0 (<packageName>)"` — пропустили.
- **Карта моргала при рекомпозиции.** Compose пересоздавал `MapView` на каждый чих. Решил через `remember { createMapView(...) }` плюс `DisposableEffect` с `onResume/onPause`.
- **Падение при потере сети.** OSRM-вызов теперь обёрнут в `runCatching`, ошибка идёт в `StateFlow<String?>` и показывается через `SnackbarHost`.

## Что бы доделал, если бы было время

- Сохранять список укрытий в Room, чтобы не потерять после перезапуска.
- Офлайн-карты: предзагрузка тайлов выбранного региона.
- Геофенсинг — уведомление, когда подошёл к укрытию.
- Свой self-hosted OSRM на VPS — публичный `router.project-osrm.org` имеет rate limit и не для продакшна.
- Кастомные иконки укрытий по типу (бомбоубежище / медпункт / точка сбора).
